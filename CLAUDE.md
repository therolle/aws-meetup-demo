## Project Overview

AWS Amplify Gen2 Todo demo app — a React + TypeScript frontend with an Amplify Gen2 backend (auth + data). Uses Cognito user pools for authentication and a DynamoDB-backed GraphQL API (owner-based authorization).

## Commands

- `npm run dev` — start Vite dev server
- `npm run build` — typecheck with `tsc` then build with Vite (output in `dist/`)
- `npm run preview` — preview production build locally
- `npx ampx sandbox` — start Amplify cloud sandbox for backend development (generates `amplify_outputs.json`)
- `npx ampx sandbox delete` — tear down sandbox resources

No test runner or linter is configured.

## Architecture

**Frontend** (`src/`): React 18 + Vite 6 + Tailwind CSS v4 (via `@tailwindcss/vite` plugin). The `Authenticator` component wraps the app in `main.tsx`, so all routes require sign-in. `App.tsx` uses `generateClient<Schema>()` for typed data operations and `observeQuery()` for real-time subscription to todos.

**Backend** (`amplify/`): Amplify Gen2 code-first backend defined in TypeScript.
- `backend.ts` — entry point, wires auth + data resources
- `auth/resource.ts` — Cognito auth with email login
- `data/resource.ts` — GraphQL schema with a single `Todo` model (owner-authorized, backed by DynamoDB)

**Data flow**: The typed `Schema` export from `amplify/data/resource.ts` is imported by the frontend for type-safe client generation. `amplify_outputs.json` (gitignored, generated by sandbox) provides runtime configuration.

## Key Conventions

- Package manager: npm (lockfile: `package-lock.json`)
- Tailwind v4 uses `@theme` in CSS for custom colors (see `src/index.css`), not `tailwind.config.js`
- Strict TypeScript (`noUnusedLocals`, `noUnusedParameters`, `noFallthroughCasesInSwitch`)
- `amplify_outputs.json` is gitignored — never commit it; it's generated by `ampx sandbox`
- CI/CD via `amplify.yml` — backend deploys with `ampx pipeline-deploy`, frontend builds with `npm run build`
